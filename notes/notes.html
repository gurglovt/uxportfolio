Research Findings

Key Insights

Convenience Over Organization

Users use whatever is at hand and convenient. They will use solutions that are impractical in the long term if they work in the instant.

Offline Support Essential

Sometimes the user doesn't have internet. Offline support with auto-sync when connection is established is crucial.

Multi-Functionality

Users tend towards solutions that serve multiple functions (e.g., calendar + reminders, notes + media transfer).

Intuitive Organization

Organization should be intuitive and automatic. The user shouldn't need to think so much.

Notes Are Ephemeral

Notes are often edited and deleted. The app should make these easy to do, while allowing easy error recovery.

Whiteboard Space Needed

It's useful to have a "whiteboard" space with immediately important information that can be easily updated.

Competitive Audit

Market Analysis

Evernote, Notion, and Google Keep

Competitive Audit

Competitor Advantages

  • Significantly refined and capable feature sets
  • Powerful search and audio transcription features
  • Evernote's "scratchpad" serves the whiteboard need
  • Notion's block system makes notes versatile and easily editable
  • Focus on minimal taps and widget integration
  • Personalization with icons and cover images (Notion)
  • Built-in calendars or calendar companion apps
  • Friendly tone with illustrations and personality touches
Competitive Audit

Competitor Disadvantages

  • Free options feel bare without customization
  • Paid options feel bloated and run slowly
  • Note organization limited to pinning (Google Keep)
  • Poor optimization causing loading times
  • Limited image support across all apps
  • Widgets don't allow editing or appending notes
  • Expensive for individuals with little free tier value
  • Hard text limits or storage restrictions
  • Poor file navigation systems (tile view only)

Define

Core Features

  • Architecture: Notepads section and Lists/Reminders section
  • Block system like Notion for flexible content
  • Whiteboard/scratchpad feature (the "Dump")
  • Keyboard-ready typing zone for quick entry
  • Options to wipe, convert to new note, or append to existing note
  • File browser-style note navigation
  • Expand/collapse all notebooks, recent sort, favorites
  • Focus on functional, offline, digital notepad

Test 1 Results

User 1 Feedback

  • Correctly identified as note-taking app

  • Identified subnote creation and navigation
  • Unclear about expand/contract all buttons
  • Assumption that bell icon sets time (incorrect)

  • Home page has a lot going on
Test 1 Results

User 2 Feedback

  • Compared to Obsidian, Miro, and Notion
  • Confused by Dump expansion arrow
  • Wants note and list functions consolidated
  • Praises organization with grey borders
  • Misinterprets expand/contract as full-screen
  • "I don't get why it's called Dump!"
  • Structure of app somewhat confusing
  • States all should be on one page
Test 1 Insights

Actions Taken

Dump Presentation

The Dump is our USP. Its presentation needs reworking as it confused users. The expand icon needs consistency.

List Consolidation

List section will be consolidated with notebooks to reduce separation. Lists can be made within notes.

Icon Clarity

Expand/contract buttons confused users. Icons need to be more intuitive or users need to discover through interaction.

Navigation Rework

Need to rework function and access between notes and notebook levels of navigation.
